# TAGS — Tiered Multi-Agent Graveyard Shift

This is the home for the **TAGS** initiative: a tiered, multi-vendor *execution architecture* for autonomous fleet work (planner/worker model-tier routing, multi-vendor budgets, escalation paths, parallel worktree fleets).

TAGS is a **separate initiative** from the Operating Model. The Operating Model is about *where truth lives* (knowledge homes / lifecycle); TAGS is about *how the fleet executes* (the engine). They are deliberately kept as their own threads and must not be entangled.
